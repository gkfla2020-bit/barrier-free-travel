# 발표 시연 시나리오 — "조금 돌아가더라도, 갈 수 없는 길이 없게"

멘토 피드백("이게 실재하는 문제인가?")에 대한 답: **같은 출발·도착인데 일반 지도앱은
이동약자가 물리적으로 갈 수 없는 경로를 안내한다**는 것을 실측 수치로 보여준다.

모든 수치는 Tmap 보행자 API 실호출 결과 (일반 = searchOption 0 / 우리 앱 = 30).
전 지역(서울 제외 9곳) 장소 쌍 ~200개를 두 옵션으로 스캔해 선정했다.

---

## 메인 시나리오: 수원 화성 — 팔달문 → 팔달공원

| | 일반 지도앱 (Tmap 기본) | 모두의 여행 (계단 회피) |
|---|---|---|
| 경로 | **672m · 9분** | 1,519m · 20분 |
| 실체 | **팔달산 계단 2~3곳 직등** — 휠체어 통행 불가 | 계단 0 — 공원 진입로 우회 |
| 결론 | 빠르지만 **갈 수 없는 길** | **+10분 돌아가지만 갈 수 있는 길** |

- 팔달문은 앱의 수원 **출발지(origin)** 라서 앱 흐름 그대로 재현된다:
  설문 → "수원" → 카드덱에서 **팔달공원 담기** → 경로 화면이 곧 오른쪽 패널.
- 왼쪽 패널은 일반 지도앱(티맵/카카오맵 도보) 스크린샷: 같은 출발·도착 검색 화면.
  ⚠️ 캡처 시 경로 상세를 펼쳐 "계단" 안내가 보이게 찍을 것.

### PPT 한 장 구성

```
[슬로건] 조금 돌아가더라도, 갈 수 없는 길이 없게

  왼쪽 (일반 지도앱 캡처)          오른쪽 (우리 앱 캡처)
  "9분 · 672m"                    "20분 · 1.5km"
  경로에 계단 3곳 ← 빨간 원 강조     계단 0 · 난이도 배지
  "휠체어는 여기서 멈춥니다"          "10분 돌아가면 도착할 수 있습니다"
```

숫자 자막: **+847m, +10분. 하지만 도착할 수 있는 유일한 경로.**

## 보조 시나리오 (같은 수원 코스에 자연스럽게 포함)

카드덱 후보권(중심 최근접 관광지 10 + 음식점 6) 안에서 검증된 쌍:

| 출발 → 도착 | 일반 | 회피 | 추가 |
|---|---|---|---|
| 행궁동 벽화마을 → 팔달공원 | 1,626m/23분 · 계단 2 | 2,372m/31분 · 계단 0 | **+7.4분** |
| 팔달공원 → 팔달제과 | 745m/10분 · 계단 2 | 1,589m/20분 · 계단 0 | +9.5분 |
| 장안공원 → 연무대 | 1,223m/16분 · 계단 3 | 1,303m/17분 · 계단 0 | **+0.8분** ← "거의 공짜 우회"도 있음 |

**추천 시연 코스**: 팔달문(출발) → 행궁동 벽화마을 → 팔달공원 → 수원통닭거리(식사)
— 전 구간 후보권 장소라 카드덱으로 만들 수 있고, 벽화마을→팔달공원 구간에서
"+7.4분 우회" 스토리가 라이브로 나온다.

## 백업: 수원역 지하보도 케이스 (다른 유형의 장벽)

팔달문 → 소롯(수원역 앞): 일반 경로가 **"수원역 8번출구 지하보도 진입 + 육교 진입"**
안내 (2,755m/41분) → 회피 시 지상 경로 (3,383m/47분, **+6.6분**).
계단이 아닌 '지하보도·육교'형 장벽 예시로 언급용. (소롯은 카드덱 후보권 밖이라
라이브 시연보다는 슬라이드 수치로 사용.)

## 발표 Q&A 대비 팩트

- 왜 서울이 아닌가 → 시연 안정성(대중교통·저상버스까지 부산/수원에서 전부 실데이터 동작).
- 수치 출처 → Tmap 보행자 API searchOption 0 vs 30, 2026-07 실호출.
- "일반 지도앱" 표현 → 티맵 도보 기본 옵션 기준임을 명시 (타사 앱도 캡처로 첨부 가능).
- 계단 감지 방법 → 경로 LineString facilityType 17(계단) — 옵션30에서 완전 소거됨을
  실측으로 확인해 역공학한 값 (공식 문서에 코드표 부재).
