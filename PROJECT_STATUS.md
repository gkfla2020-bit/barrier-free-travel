# 편해질지도 — 프로젝트 현황 및 기획 팔로업 문서

> **한 줄 소개:** 이동약자(휠체어 이용자·고령자·유모차 동반 가족)가 "이 길로 가도 될까?"를 고민하지 않고 여행을 계획할 수 있게 돕는 무장애 여행 플래너.
>
> 이 문서는 **기획팀이 개발 상황을 기술 용어 없이 따라올 수 있도록** 작성되었습니다.
> "무슨 기능이 있고 / 왜 필요했고 / 어떻게 동작하고 / 어떤 문제를 어떻게 풀었는지"를 순서대로 설명합니다.
> 최종 업데이트: 개발 진행에 따라 계속 갱신됩니다.

## 1. 프로젝트가 풀려는 문제

일반적인 지도·길찾기 앱은 "가장 빠른 길"을 알려줍니다. 하지만 휠체어 이용자에게 "가장 빠른 길"은 계단을 포함할 수 있고, 그러면 **아예 갈 수 없는 길**이 됩니다.

이동약자가 여행을 계획할 때 실제로 겪는 어려움:

- **"이 관광지에 휠체어로 들어갈 수 있나?"** — 정보가 흩어져 있어 일일이 검색해야 함
- **"가는 길에 계단이 있나?"** — 일반 길찾기는 계단을 그냥 안내함
- **"버스가 오면 탈 수 있나?"** — 저상버스가 아니면 휠체어로 못 탐
- **"중간에 화장실 쓸 수 있나?"** — 장애인 화장실 위치를 미리 알기 어려움

**편해질지도는 이 4가지 불안을 한 앱에서 해소하는 것**을 목표로 합니다.

## 2. 핵심 콘셉트

**"편해질지도"** = "편해질지도(모른다)" + "지도(map)"의 중의적 이름.
→ *이동약자의 여행이 편해질 수 있다는 희망* + *실제로 길을 안내하는 지도* 를 동시에 담았습니다.

핵심 가치는 **"안전을 위해 보수적으로 판단한다"**입니다.
- 데이터로 확인할 수 없는 요소(예: 계단 칸수, 승강기 유무)는 **"괜찮겠지"가 아니라 "위험할 수 있음"으로 처리**합니다.
- 잘못된 "접근 가능" 표시는 이동약자에게 치명적이기 때문입니다.

## 4. 기능별 상세 설명

### 4-1. 무장애 장소 표시
- **무엇:** 전국 10개 지역의 관광지·음식점 342곳을 지도에 표시합니다.
- **접근성 배지:** 각 장소마다 4종의 배지를 붙입니다 — 휠체어 / 엘리베이터 / 장애인 화장실 / 장애인 주차.
- **어떻게 판단하나:** 한국관광공사의 무장애 여행정보를 읽어, "있음/가능/설치"처럼 긍정 표현이 있고 "없음/불가"처럼 부정 표현이 없을 때만 배지를 답니다. 애매하면 배지를 안 답니다(보수적 판단).
- **지도 필터:** 상단에서 "휠체어", "장애인 화장실" 등으로 필터링하면 해당 시설이 있는 곳만 지도에 남습니다.

### 4-2. 맞춤 코스 추천
- **무엇:** 사용자의 이동 조건(페르소나)에 맞는 장소만 골라 자동으로 여행 코스를 만듭니다.
- **흐름:** 설문(휠체어/유모차 등 + 필수 시설 선택) → 조건을 만족하는 후보를 카드로 제시 → 마음에 드는 곳을 담으면 → 가까운 순서로 자동 정렬해 코스 완성.
- **동선 최적화:** '여유롭게' 코스를 고르면 서로 가까운 장소끼리 묶어 이동 거리를 짧게 만듭니다.
- **AI 추천:** 채팅으로 "휠체어로 반나절 코스"처럼 요청하면 AI(Claude)가 조건에 맞는 코스를 제안합니다.

### 4-3. 계단 회피 도보 경로
- **무엇:** 목적지 간 도보 경로를 '계단 제외'를 우선으로 안내합니다.
- **난이도 표시:** 경로를 쉬움 / 중간 / 어려움으로 표시하고, 그 이유(계단 N회, 경사로, 횡단보도 등)를 함께 보여줍니다.
- **판단 원칙:** 경로에서 **가장 어려운 요소 하나**가 전체 난이도를 결정합니다. 육교·지하보도처럼 승강기 유무를 확인할 수 없는 곳은 안전하게 '어려움'으로 봅니다.

### 4-4. 출발지 선택 (다중) — *최근 확장됨*
- **무엇:** 코스를 만들기 전에 "어디서 출발할지"를 고릅니다.
- **개선 내용:** 기존에는 지역당 출발지가 2개뿐이었는데, **3~4개로 늘렸습니다** (전국 총 32곳). 주요 지하철역·버스터미널·주요 주차장을 포함합니다.
  - 예) 서울: 광화문역 / 서울역 / 시청역 / 종로3가역
- **내 위치 출발:** GPS로 현재 위치가 해당 지역 안에 있으면 "내 위치"도 출발지로 선택 가능합니다.
- **더 보기:** 출발지가 많으면 처음 3개만 보이고 "더 보기"로 나머지를 펼칩니다.

### 4-5. 채팅으로 출발지 인식 — *최근 추가됨*
- **무엇:** 버튼을 누르지 않고 **대화로 출발지를 정할 수 있습니다.**
- **예시:**
  - "광화문역에서 출발" → 광화문역으로 설정 ✅
  - "시청에서 시작할게" → 시청역으로 인식 (줄임말·조사 처리) ✅
  - "서울역" → 서울역으로 설정 ✅
- **똑똑한 처리:**
  - "시청"만 말해도 "시청역"으로 알아듣습니다 (역/터미널 같은 접미사 자동 처리).
  - 여러 곳이 매칭되면 "여러 출발지가 매칭돼요: ○○, △△. 하나만 말씀해주세요"라고 되물어봅니다.
  - 해당 지역에 없는 곳을 말하면 "이 지역에서 고를 수 있는 출발지는 ○○입니다"라고 안내합니다.
- **버튼과 완전 연동:** 채팅으로 고르든 버튼으로 고르든 동일하게 동작하고, 이미 코스가 있으면 새 출발지 기준으로 경로를 다시 계산합니다.

### 4-6. 대중교통 경로
- **무엇:** 선택한 출발지에서 코스 장소까지 **버스·지하철 경로**를 안내합니다.
- **포함 정보:** 예상 소요시간, 도보 구간 거리, 어느 노선을 타는지, 몇 정거장인지.
- **이동약자 우선 원칙:** 가장 빠른 경로가 아니라 **환승이 적은 경로 → 도보가 적은 경로**를 우선합니다. 이동약자에게 환승 1회는 시간 절약보다 부담이 크기 때문입니다.
- **도보/대중교통 전환:** "도보만" / "대중교통 포함" 버튼으로 두 방식을 비교할 수 있습니다.

### 4-7. 실제 도로 형상 표시 — *최근 수정됨*
- **문제였던 것:** 예전에는 버스 노선이 정류장끼리 **직선으로** 이어져 건물을 가로지르는 것처럼 보였습니다.
- **해결:** 실제 도로 형상 데이터를 받아 노선을 **도로 모양 그대로** 그립니다. (예: 402번 버스 = 예전 9개 점 직선 → 실제 도로 115개 점)
- **안전장치:** 실제 형상을 못 받는 경우엔 "정류장 간 개략 직선입니다"라고 명시하고 정류장 위치에 마커를 찍어, 사용자가 오해하지 않게 합니다.

### 4-8. 정류장 이름 표시 — *최근 추가됨*
- **무엇:** 대중교통 구간에서 **어느 정류장/역에서 타고 내리는지** 이름으로 보여줍니다.
- **지도:** 정류장마다 마커가 찍히고, 클릭하면 정류장 이름이 뜹니다.
- **경로 안내:** "정류장 9개 보기"를 누르면 승차→하차까지 거치는 모든 정류장 이름을 순서대로 확인할 수 있습니다.

### 4-9. 저상버스 실시간 안내
- **무엇:** 다음에 오는 버스가 **저상버스(휠체어 탑승 가능)인지 실시간으로** 조회해 알려줍니다.
- **안내:** 저상버스면 "다음 ○○번은 저상버스입니다 · 약 N분 후 도착", 일반 버스면 "다음 차를 기다리는 것을 권합니다"라고 안내합니다.
- **데이터 출처:** 전국 버스정보(TAGO) + 서울 버스정보시스템(BIS)을 함께 사용합니다.

### 4-10. 장애인 화장실 커버리지
- **무엇:** 추천 코스의 **각 장소 주변 500m 이내** 접근 가능한 장애인 화장실을 자동으로 찾아 표시합니다.
- **표시:** "가장 가까운 접근 화장실: ○○ (320m)" 또는 "주변 인증 화장실 없음".
- **데이터 공백 보강:** 화장실 정보가 부족한 지역은 별도로 데이터를 채워, 어느 지역을 골라도 최소한의 화장실 정보를 제공합니다.

## 5. 우리가 마주친 문제와 해결 과정

기획팀이 "왜 이런 결정을 했는지" 이해할 수 있도록, 개발 중 마주친 실제 문제와 해결을 정리합니다.

### 문제 ① 무장애 장소 정보가 흩어져 있다
- **상황:** 이동약자가 갈 만한 곳을 찾으려면 여러 사이트를 뒤져야 했음.
- **해결:** 한국관광공사 무장애 여행정보를 미리 수집해 지도에 배지와 함께 통합 표시.

### 문제 ② 일반 길찾기는 계단을 안내한다
- **상황:** 휠체어 이용자에게 계단 경로는 무의미.
- **해결:** '계단 제외' 우선 도보 경로 + 계단·경사·육교를 감지해 난이도로 경고. 확인 불가 요소는 안전 우선으로 판정.

### 문제 ③ 출발지에 따라 경로가 달라야 한다
- **상황:** 어디서 출발하는지에 따라 대중교통·도보가 완전히 달라짐.
- **해결:** 지역별 검증된 출발지를 제공하고, 출발지를 먼저 고르게 함. → 이후 **출발지 다중화 + 채팅 인식**으로 확장 (4-4, 4-5).

### 문제 ④ 버스·지하철로 어떻게 가는지 모른다
- **상황:** 대중교통 경로 안내가 없었음.
- **해결:** 대중교통 경로 + 예상 시간 + 도보 구간 안내. 환승 최소 → 도보 최소 원칙 적용.

### 문제 ⑤ 지도의 버스 노선이 건물을 가로지른다
- **상황:** 노선이 정류장 직선으로 그려져 실제 도로를 무시.
- **해결:** 실제 도로 형상 데이터를 받아 도로 모양대로 표시. 못 받으면 "개략 직선"임을 명시.

### 문제 ⑥ 일반 버스가 오면 휠체어로 못 탄다
- **상황:** 저상버스가 아니면 탑승 불가.
- **해결:** 실시간으로 다음 버스의 저상 여부를 조회해 안내. (+ 지방 저상버스 조회가 안 되던 문제도 팀에서 수정)

### 문제 ⑦ 이동 중 화장실을 쓸 수 있을지 불안하다
- **상황:** 장애인 화장실 위치를 미리 알기 어려움.
- **해결:** 코스 장소 주변 500m 화장실 자동 탐색 + 데이터 공백 지역 보강.

### 문제 ⑧ 정류장 직선을 실제 길로 못 바꾸면?
- **상황:** 실제 도로 형상 데이터를 못 받는 경우 대비 필요.
- **해결(기획적 결정):** 억지로 직선을 그려 오해를 주기보다, **"정류장 to 정류장 개략 직선"임을 솔직히 밝히고** 정류장 마커로 표시. → 정직한 정보 전달을 택함.

## 6. 사용자 여정 (실제 사용 흐름)

```
1) 설문        이동 조건 선택 (휠체어 / 고령자 / 유모차, 필수 시설)
      ↓
2) 지역 선택   채팅으로 "서울", "경주" 등 입력
      ↓
3) 출발지 선택  버튼으로 고르거나, 채팅으로 "광화문역에서 출발"
      ↓
4) 후보 카드    조건에 맞는 장소를 카드로 넘기며 담기
      ↓
5) 자동 코스    담은 장소를 가까운 순으로 정렬해 코스 완성
      ↓
6) 경로 확인    도보 / 대중교통 전환, 지도에 노선·정류장·화장실 표시
```

각 단계는 채팅과 버튼 UI 양쪽으로 진행할 수 있어, 사용자가 편한 방식을 섞어 쓸 수 있습니다.

## 7. 지원 지역 및 데이터 현황

- **지원 지역 (10곳):** 서울 · 경주 · 부산 · 전주 · 강릉 · 여수 · 제주 · 수원 · 인천 · 대구
- **장소 데이터:** 관광지·음식점 342곳 (접근성 배지 포함)
- **출발지:** 지역당 3~4곳, 전국 32곳 (지하철역/버스터미널/주차장)
- **화장실:** 각 지역 최소 1곳 이상 보장 (부족한 지역은 보강 완료)

## 9. 진행 예정 / 백로그

현재 완료된 기능 외에, 논의 가능한 다음 후보들입니다. (우선순위는 기획팀과 협의 필요)

- **코스 저장 / 공유** — 만든 여행 코스를 저장하거나 링크로 공유
- **출발지 더 확대** — 랜드마크·병원·숙소 등 출발지 후보 추가
- **접근성 정보 실시간화** — 현재는 미리 수집한 데이터, 향후 실시간 조회 검토
- **디자인 고도화** — 브랜드 색상·아이콘 등 시각 완성도 향상
- **다국어 지원** — 외국인 이동약자 관광객 대응

## 10. 용어 사전

기획팀이 개발 문서를 볼 때 참고할 수 있는 용어 정리입니다.

| 용어 | 뜻 |
|------|-----|
| 이동약자 | 휠체어 이용자, 고령자, 유모차 동반 가족 등 이동에 제약이 있는 사람 |
| 무장애(배리어프리) | 장애물이 없어 누구나 이용할 수 있는 환경 |
| 저상버스 | 바닥이 낮고 경사판이 있어 휠체어가 탈 수 있는 버스 |
| 페르소나 | 사용자의 이동 조건 프로필 (예: 휠체어 + 화장실 필수) |
| 코스 | 사용자가 방문할 장소들의 순서 있는 목록 |
| 배지 | 장소의 접근성 정보 아이콘 (휠체어/엘리베이터/화장실/주차) |
| 커버리지 | 코스 주변에 화장실이 얼마나 확보되는지 |
| 폴백(fallback) | 원래 방식이 안 될 때의 대체 동작 (예: 실제 도로 못 받으면 직선 표시) |
