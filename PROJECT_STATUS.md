> 이 문서는 기획팀이 현재 개발 상황을 따라올 수 있도록, **무슨 기능이 있고 / 어떤 문제를 / 어떻게 풀었는지**를 정리합니다.

## 2. 우리가 푼 문제들 (문제 → 해결)

### 문제 1. "무장애 장소가 어디 있는지 흩어져 있어 찾기 어렵다"
- **해결:** 한국관광공사 무장애 여행정보(TourAPI)를 미리 수집해 지도에 배지(휠체어/화장실 등)와 함께 표시. 전국 10개 지역 342곳.

### 문제 2. "일반 길찾기는 계단을 안내해서 휠체어로 못 간다"
- **해결:** Tmap 보행자 API의 '계단 제외' 옵션을 우선 사용. 계단·육교·경사로를 감지해 난이도(쉬움/중간/어려움)로 표시하고, 확인 불가한 요소는 안전하게 '어려움'으로 판정.

### 문제 3. "어디서 출발하는지에 따라 경로가 달라야 한다"
- **해결:** 지역마다 검증된 고정 출발지 2개(지하철역/버스터미널/주차장)를 제공하고, 코스 생성 전에 사용자가 출발지를 고르게 함. GPS '내 위치'도 선택 가능.

### 문제 4. "대중교통(버스·지하철)으로 어떻게 가는지 모른다"
- **해결:** ODsay 길찾기로 지하철·버스 경로 + 예상 시간 + 도보 구간을 안내. 이동약자 원칙상 '환승 최소 → 도보 최소' 순으로 경로를 고름.

### 문제 5. "지도의 버스 노선이 건물을 가로지르는 직선으로 그려진다"
- **해결:** ODsay loadLane API로 실제 도로 형상 좌표를 받아 노선을 도로 모양대로 그림. (예: 402번 버스 = 정류장 9점 직선 → 실제 도로 115점) 실제 형상을 못 받으면 "정류장 간 개략 직선"임을 명시하고 정류장 마커를 표시.

### 문제 6. "일반 버스가 오면 휠체어로 못 탄다"
- **해결:** TAGO(전국)+서울 BIS 실시간 API로 다음 버스가 저상버스인지 조회해 안내. 일반차량이면 "다음 차를 기다리세요" 안내.

### 문제 7. "이동 중 화장실을 쓸 수 있을지 불안하다"
- **해결:** 추천 코스의 각 장소 주변 500m 내 접근 가능한 장애인 화장실을 자동으로 찾아 이름·거리와 함께 표시. 화장실 데이터가 부족한 지역은 별도로 보강.

## 3. 현재 사용자 흐름

1. **설문** — 이동 조건 선택 (휠체어/유모차 등, 필수 시설)
2. **지역 선택** — 채팅으로 "서울", "경주" 등 입력
3. **출발지 선택** — 지역별 출발지 2개 또는 내 위치 중 선택
4. **후보 카드** — 조건에 맞는 장소를 카드로 넘기며 담기
5. **자동 코스** — 담은 장소를 가까운 순으로 정렬해 코스 생성
6. **경로 확인** — 도보 / 대중교통 전환, 지도에 노선·정류장·화장실 표시

## 5. 진행 예정 (다음 작업)

### 출발지 다중화 + 채팅 인식
- **문제:** 지금은 출발지가 지역당 2개로 제한적이고, 버튼으로만 고를 수 있다.
- **목표:**
  - 출발지 후보를 더 늘린다 (주요 역·터미널·랜드마크).
  - 채팅으로 "광화문역에서 출발", "시청에서 시작" 같은 문장을 인식해 출발지를 자동 설정.
- **상태:** 스펙 작성 후 개발 예정.
